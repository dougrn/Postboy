# Roadmap - Postboy

## Phase 1: Core Stability (Completed)
- [x] Basic CRUD for Collections/Environments.
- [x] Request execution with variable replacement.
- [x] UI/UX improvements (Glassmorphism).
- [x] Custom Modal system.
- [x] Postman Import.

## Phase 2: Power User Features (Completed)
- [x] **Scripting Support**: Pre-request and Post-request JS scripts.
- [x] **Tests**: Basic assertion engine for response status/body.
- [x] **Code Editor**: Monaco Editor integration with syntax highlighting.
- [x] **Developer Console**: Sandboxed JS execution environment.
- [x] **Interactive JSON Viewer**: Expandable/collapsible JSON trees via jsoneditor.

## Phase 3: Advanced Protocols & Organization (Completed)
- [x] **WebSocket Client**: Native support for WS connections with message history.
- [x] **GraphQL Support**: Query execution with variables.
- [x] **File Uploads**: Multipart/form-data support with Base64 encoding.
- [x] **Nested Folders**: Hierarchical organization inside collections.
- [x] **Tab Interface**: Persistent tabs with keyboard shortcuts.

## Phase 4: Advanced Integration (Upcoming)
- [ ] **Visualizers**: Tab for rendering HTML/Charts from responses.
- [ ] **CURL Support**: Import/Export via CURL commands.
- [ ] **OAuth2**: Integrated OAuth2 flow helper.
- [ ] **gRPC Support**: Native protocol buffers communication.
- [ ] **Mock Server**: Built-in endpoints for frontend testing.

## Phase 5: Developer Experience
- [ ] **Sync**: Optional local network sync between devices.
- [ ] **CLI**: Command line interface for running collections in CI/CD.
- [ ] **Plug-in System**: Allow custom body parsers.
- [ ] **Themes**: More color presets beyond default Dark Mode.
